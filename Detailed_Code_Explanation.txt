========================================================================
AI-POWERED RESUME GRADER: DETAILED CODE EXPLANATION
========================================================================

This document provides a comprehensive, file-by-file explanation of the complete codebase for both the Backend and Frontend of the AI-Powered Resume Grader application.

------------------------------------------------------------------------
PART 1: BACKEND DIRECTORY (`/backend`)
------------------------------------------------------------------------
The backend is built with Node.js and Express. It receives PDF files, extracts the text, sends the text to the Hugging Face AI API for analysis, and stores records in MongoDB.

1. `server.js` (Root Directory)
   - This is the main entry point of the backend application.
   - It initializes the Express application, sets up CORS (to allow the frontend to communicate with it), and loads environment variables using `dotenv`.
   - It calls `connectDB()` from the config folder to establish a connection to MongoDB.
   - It mounts the resume-related API routes under the `/api/resume` prefix.
   - It starts the server listening on the port defined in `.env`.

2. `config/db.js`
   - Contains the `connectDB` asynchronous function.
   - It uses Mongoose to connect to the MongoDB instance defined by the `MONGO_URI` environment variable.
   - If the connection fails, it catches the error, logs it to the console, and exits the process to prevent the app from running without a database.

3. `models/Resume.js`
   - Defines the Mongoose Schema for the `Resume` database collection.
   - It dictates what data is saved for each uploaded resume (e.g., filename, extracted text, AI generated score, extracted keywords, missing keywords, and suggestions).
   - This structure directly maps to the JSON response provided by the Hugging Face AI.

4. `routes/resumeRoutes.js`
   - Defines the API endpoints for the application.
   - Exposes a `POST /upload` route.
   - It chains two important things to this route: first, the `uploadMiddleware.single('resume')` which handles the incoming file, and second, the `resumeController.uploadResume` function which executes the business logic.

5. `middlewares/uploadMiddleware.js`
   - Configures `multer`, a library for handling `multipart/form-data`.
   - It sets up a storage engine (typically memory storage) so the uploaded PDF is available as a buffer (`req.file.buffer`) rather than being saved to disk.
   - It includes a file filter to ensure only `.pdf` files are accepted.

6. `controllers/resumeController.js`
   - The core brain of the backend's request lifecycle.
   - When a file is uploaded, it verifies the file exists.
   - It passes the file buffer to `pdfService.js` to extract plain text.
   - It then takes that plain text and passes it to `aiService.js` for Hugging Face analysis.
   - Once the analysis is returned, it structures the response, optionally saves the record to MongoDB using the `Resume` model, and sends the JSON result back to the frontend.

7. `services/pdfService.js`
   - A helper service that uses the `pdf-parse` library.
   - It takes the raw buffer of the PDF file and extracts all readable text from it.

8. `services/aiService.js`
   - Handles the integration with the Hugging Face API.
   - Uses the `@huggingface/inference` library to construct a prompt detailing that it is an ATS system and requesting a strict JSON response.
   - It calls the `Qwen/Qwen2.5-72B-Instruct` model (or similar) with the prompt and the extracted resume text.
   - It parses the AI's response to ensure it's valid JSON and returns it to the controller.

9. `test_huggingface.js`
   - A standalone utility script to verify that the `HUGGINGFACE_API_KEY` is active and working.
   - It can be run independently using `node test_huggingface.js` for debugging purposes.


------------------------------------------------------------------------
PART 2: FRONTEND DIRECTORY (`/frontend`)
------------------------------------------------------------------------
The frontend is built with React and Vite. It provides the user interface for uploading resumes and visualizing the AI's feedback.

1. `src/main.jsx` & `src/App.jsx`
   - `main.jsx` is the React entry point that mounts the application to the DOM.
   - `App.jsx` sets up the React Router. It defines the routes for the application, such as the `/` (Home) page and potentially a `/dashboard` or `/results` page.

2. `src/apis/api.js`
   - Contains the Axios configuration.
   - Defines the `uploadResume` function, which creates a `FormData` object, appends the selected PDF file, and makes a POST request to `http://localhost:5000/api/resume/upload`.

3. `src/pages/Home.jsx`
   - The main landing page of the application.
   - It houses the `FileUpload` component and manages the primary state of the application (e.g., loading state, error messages, and the final result data).
   - Once a file is successfully uploaded and graded, it either displays the results directly or redirects to the Dashboard.

4. `src/pages/Dashboard.jsx`
   - The layout page responsible for cleanly presenting the analysis results.
   - It receives the AI grading data (Score, Keywords, Suggestions) and renders the appropriate child components to visualize this data.

5. `src/components/FileUpload.jsx`
   - A drag-and-drop file upload interface.
   - It handles user interactions like dragging a file over the zone, selecting a file via the file browser, and validating that the selected file is indeed a PDF.
   - It triggers the upload process when a file is selected.

6. `src/components/ScoreDisplay.jsx`
   - A visual component dedicated to displaying the overall ATS score (0-100) returned by the AI.
   - It typically includes visual indicators like a circular progress bar or color-coding (red for low, green for high) to give immediate feedback.

7. `src/components/KeywordList.jsx`
   - A reusable component that maps through arrays of keywords.
   - Used to display both the `keywordsExtracted` (skills the user has) and `missingKeywords` (skills the user should add).
   - Often styled as small tags or pills for a modern aesthetic.

8. `src/index.css` & `src/App.css`
   - Global and component-level CSS files.
   - They contain all the styling for the application, ensuring a modern, responsive, and visually appealing user interface.
